Application Overview

The InfoArchive Data Transformer is a sophisticated enterprise application designed to automate the complex process of transforming CSV data into InfoArchive-compatible XML format. It seamlessly integrates with Apache Airflow 3.x for robust workflow orchestration and provides an intuitive web interface for managing data transformation pipelines.

CSV Upload & Validation

Intelligent file processing with automatic structure analysis and comprehensive validation.

Schema Generation

Automatic InfoArchive-compatible schema creation with intelligent field mapping.

Visual Monitoring

Real-time workflow tracking and analytics with interactive dashboards.

Enterprise Reporting

Comprehensive audit trails and performance metrics with export capabilities.

Detailed Step-by-Step Workflows

1

Prerequisites & Source Data Upload

Goal: Upload and validate your CSV files for processing
Navigation: Sidebar → Structured Data Archival → Prerequisites & Source Data Upload

What You'll See:

  • Clean, intuitive file upload interface
  • Drag-and-drop area for CSV files
  • Real-time file validation messages
  • Interactive preview of uploaded data with headers and sample rows
  • File size and format validation indicators

Actions to Take:

  1. Click "Choose File" or drag your CSV file directly to the upload area
  2. Wait for automatic validation - the system will analyze file structure and content
  3. Review the data preview - verify headers are correct and sample data looks accurate
  4. Check validation messages - ensure no errors are reported
  5. Click "Continue" to proceed to schema configuration

Success Indicators:

  • File successfully uploaded without errors
  • CSV headers detected and properly displayed
  • Sample data rows shown in organized preview table
  • File size and format validation passed
  • "Continue" button becomes active and clickable
2

Generate Table Schema in InfoArchive

Goal: Create InfoArchive-compatible schema from your CSV structure
Navigation: Automatically advances from Step 1, or manually via sidebar menu

What You'll See:

  • Comprehensive schema generation interface
  • Field mapping table with source-to-target assignments
  • Automatic data type detection and assignment
  • InfoArchive compatibility validation checks
  • Preview of generated schema structure

Actions to Take:

  1. Review auto-detected fields from your uploaded CSV file
  2. Verify field mappings - ensure each CSV column maps correctly
  3. Check data type assignments - confirm automatic type detection is accurate
  4. Click "Analyze" to generate the complete InfoArchive schema
  5. Wait for processing - DAG 'IA_CSV_Metadata_Generator' will be triggered automatically
  6. Verify success notification confirming schema generation completion

Success Indicators:

  • DAG 'IA_CSV_Metadata_Generator' triggered successfully
  • All schema fields mapped correctly with proper naming
  • Data types assigned appropriately for InfoArchive compatibility
  • No validation errors or compatibility warnings
  • Success toast notification appears confirming completion
3

Validate and Finalize Schema

Goal: Review, edit, and finalize the generated schema with custom adjustments
Navigation: Automatically advances from Step 2 upon successful schema generation

What You'll See:

  • Interactive, editable schema interface
  • Field-by-field configuration options
  • Dropdown selectors for data types
  • Validation rules and constraints setup
  • Real-time schema preview and validation

Actions to Take:

  1. Review each field in the generated schema carefully
  2. Edit field properties as needed:
    • Field names and display labels
    • Data types (String, Integer, Date, etc.)
    • Length constraints and precision
    • Required/optional field settings
    • Default values where applicable
  3. Add validation rules if required for data quality
  4. Test the schema using the preview functionality
  5. Click "Finalize" when satisfied with all schema configurations

Success Indicators:

  • All fields properly configured with appropriate data types
  • No validation errors or schema inconsistencies
  • Schema validation passes all InfoArchive compatibility checks
  • Required fields and constraints properly defined
  • Schema marked as finalized with confirmation notification
4

Download IA Schema

Goal: Download the completed InfoArchive schema and prepare for deployment
Navigation: Automatically advances from Step 3 upon successful schema finalization

What You'll See:

  • Comprehensive download interface
  • Detailed schema summary with all configurations
  • Generated XML file preview
  • Step-by-step deployment instructions
  • Integration guidelines for InfoArchive

Actions to Take:

  1. Review the complete schema summary - verify all configurations are correct
  2. Preview the XML structure - ensure the generated format meets requirements
  3. Click "Download Schema" to save the XML file to your local system
  4. Read deployment instructions - understand integration requirements
  5. Save the file in an appropriate location for InfoArchive deployment
  6. Proceed to data transformation when ready to process actual data

Success Indicators:

  • Schema XML file successfully downloaded
  • File contains valid InfoArchive-compatible XML structure
  • All field definitions and constraints properly included
  • Deployment instructions clearly understood
  • Ready to proceed to data transformation phase
5

Transform CSV Data to IA XML Format

Goal: Transform your actual CSV data using the finalized schema and upload to InfoArchive
Navigation: Automatically advances from Step 4, or accessible via sidebar menu

What You'll See:

  • Advanced data transformation interface
  • Real-time progress indicators and status updates
  • Configurable transformation settings
  • Direct upload options to InfoArchive system
  • Error handling and retry mechanisms

Actions to Take:

  1. Configure transformation settings:
    • Batch size for processing large files
    • Error handling preferences
    • Output format specifications
  2. Click "Start Transformation" to begin the data processing
  3. Monitor progress in real-time using the progress indicators
  4. Review transformation results and handle any data quality issues
  5. Validate output format before uploading to InfoArchive
  6. Click "Upload to InfoArchive" when transformation completes successfully

Success Indicators:

  • Data transformation completed without errors
  • All CSV records successfully converted to XML format
  • Output validates against the finalized schema
  • No data quality issues or validation errors
  • Data successfully uploaded to InfoArchive system
  • Completion notification with summary statistics

Troubleshooting Guide

Common Issues & Solutions

Connection Issues

Problem: Red "Airflow Disconnected" status indicator in top-right corner

Solutions:
  1. Check if Airflow is running by navigating to http://localhost:8083
  2. Click the "Retry Connection" button in the connection warning banner
  3. Verify network connectivity and firewall settings
  4. Confirm Airflow credentials are correct (default: admin/admin123)
  5. Restart the Airflow service if necessary

File Upload Problems

Problem: CSV upload fails, shows validation errors, or file is rejected

Solutions:
  1. Verify file format is valid CSV with proper structure
  2. Check file size - ensure it's within reasonable limits (typically under 50MB)
  3. Ensure CSV has proper headers in the first row
  4. Remove special characters, accents, or symbols from column headers
  5. Verify file encoding is UTF-8 compatible
  6. Check for empty rows or malformed CSV structure

Schema Generation Errors

Problem: "Analyze" button fails or produces incorrect/incomplete schema

Solutions:
  1. Verify CSV headers are valid field names (no spaces, special characters)
  2. Check for special characters or non-standard encoding in data
  3. Ensure consistent data types within each column
  4. Review data preview for anomalies or unexpected values
  5. Try with a smaller sample file to isolate issues
  6. Check Airflow DAG logs for detailed error information

Transformation Failures

Problem: Data transformation step fails or produces errors

Solutions:
  1. Verify schema is properly finalized before transformation
  2. Check input data quality and consistency
  3. Review transformation logs in the Jobs section
  4. Ensure sufficient system resources (memory, disk space)
  5. Try processing smaller batches if dealing with large files
  6. Validate InfoArchive connectivity and permissions

Emergency Procedures

System Recovery Steps

  1. Complete Restart: Stop application → Restart Airflow → Restart application
  2. Clear Cache: Use browser developer tools to clear application cache
  3. Reset Workflow: Use "Start Over" button to reset current session
  4. Data Recovery: Check processed data directories for recovered files
  5. Connection Reset: Click "Retry Connection" or restart services
